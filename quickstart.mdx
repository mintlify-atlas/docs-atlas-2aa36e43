This page walks you through everything you need to go from zero to a working overlay. Pick the installation option that matches your setup, then follow the steps to calibrate the minimap and launch the tracker.

## Installation options

<Tabs>
  <Tab title="Pre-built package (recommended)">
    The pre-built AI 2.0 package requires no Python installation, no PyTorch setup, and no driver configuration. Download the archive, extract it, and run the executable directly.

    **Download:** [AI 2.0 pre-built package](https://github.com/761696148/Game-Map-Tracker)

    This package includes both SIFT and AI tracking modes, the minimap selector tool, and all required dependencies bundled into a single folder.
  </Tab>
  <Tab title="Source code setup">
    Running from source requires Python 3.9 or later. Clone the repository and install dependencies with pip.

    ```bash
    git clone https://github.com/761696148/Game-Map-Tracker.git
    cd Game-Map-Tracker
    pip install -r requirements.txt
    ```

    The `requirements.txt` file installs the following packages:

    ```text
    opencv-python
    numpy
    mss
    Pillow
    torch      # only needed for main_ai.py — installs CPU version by default; install the CUDA build separately for GPU support
    kornia     # only needed for main_ai.py
    requests
    ```

    <Note>
      The default `pip install torch` command downloads the CPU-only build. If you want to run AI mode with GPU acceleration, install the CUDA-enabled build from [pytorch.org](https://pytorch.org/get-started/locally/) before running `pip install -r requirements.txt`, or install it separately afterwards.
    </Note>
  </Tab>
</Tabs>

## Setup steps

<Steps>
  <Step title="Install dependencies">
    If you are using the pre-built package, skip this step — dependencies are already bundled.

    If you are running from source, install the Python packages listed in `requirements.txt`:

    ```bash
    pip install -r requirements.txt
    ```

    For AI mode with GPU support, install the CUDA build of PyTorch first. See the [PyTorch installation guide](https://pytorch.org/get-started/locally/) for the correct command for your CUDA version.
  </Step>
  <Step title="Prepare map files">
    Place two map image files in the same directory as the tracker scripts (or the pre-built executable folder):

    - `big_map.png` — a clean, annotation-free version of the full game map. This is used for feature matching.
    - `big_map-1.png` — the same map with any resource markers or overlays you want displayed in the overlay window.

    Both files must have identical pixel dimensions. The tracker will refuse to start if the sizes do not match.

    <Note>
      If you only have one map image, you can use the same file for both by copying it and naming the copy `big_map-1.png`. Tracking accuracy will be unaffected; you just won't see any extra markers in the overlay.
    </Note>
  </Step>
  <Step title="Calibrate the minimap region">
    The tracker needs to know where your game's minimap appears on screen. On first launch, the selector tool opens automatically to help you draw a capture box over the minimap.

    If you need to recalibrate later, delete or reset the `MINIMAP` section in `config.json` and relaunch, or follow the instructions in [Minimap calibration](/setup/minimap-calibration).

    The selector saves the minimap coordinates (`top`, `left`, `width`, `height`) to `config.json` when you confirm your selection.

    <Note>
      The selector tool (`selector.py`, or `MinimapSetup.exe` in the pre-built package) launches automatically on first run. You do not need to run it separately.
    </Note>
  </Step>
  <Step title="Launch the tracker">
    Choose the mode that matches your hardware and tracking needs.

    **SIFT mode** — fast, CPU-only, works on any machine:

    ```bash
    python main_sift.py
    ```

    **AI mode** — more robust, GPU-recommended, handles difficult terrain:

    ```bash
    python main_ai.py
    ```

    The floating overlay window appears immediately. Move around in your game and your position updates in real time on the map.

    <Tip>
      Not sure which mode to use? Start with SIFT. It works well for most games and regular terrain. Switch to AI mode only if SIFT loses tracking frequently due to large featureless areas or heavy UI overlays on the minimap. See [Tracking modes](/modes) for a full comparison.
    </Tip>
  </Step>
</Steps>
